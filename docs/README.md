# Documentation structure

This folder is the GitHub Pages source for the Energy Domain Ontology.

## Public website

- `index.md` is the GitHub Pages entry point.
- `site/` contains editorial website pages.
- `ontology/` contains published ontology artefacts, including static renderings and future interactive viewers.

## Documentation areas

- `guides/` — explanatory modelling guides.
- `references/` — technical reference material.
- `alignments/` — alignment reports and curation files.
- `data-contracts/` — IFC data contract documentation.
- `development/` — development notes, templates, experiments, tests, and local build instructions.

The root of `docs/` should stay small. New documentation should usually go into one of the folders above instead of being added directly to the root.
